# Retro — v1.7.0: second opinion (ROS-155)

**Shipped:** 2026-07-06 (PR [#301](https://github.com/firatcand/roster/pull/301), single-feature release)
**Scope:** `roster second-opinion` cross-model review verb + `/second-opinion` skill; scoped `claude -p` audit exception (ADR-0002).

## What happened

Ported the *idea* of forge's `/second-opinion` (fresh-process, cross-model,
honest reviewer) without its orchestrator coupling. The feature was built in a
single day under a Codex-gated loop: implement → Codex reviews the full diff
and scores /10 → fix everything real → repeat. Nine rounds
(6→7→6→7→6→8→7→7→9), 16 findings, all adopted with regression tests.

## What went well

- **The tool reviewed itself.** From round 2 the gate dispatched through the
  freshly built `roster second-opinion --diff origin/main --host codex`.
  Sentinel parsing, stdin delivery, and the codex preflight all got continuous
  real-world exercise before shipping.
- **The process caught its own worst idea.** The original plan relaxed the
  `claude -p` ban globally; Codex's round-1 block reversed it into the scoped,
  preflight-guarded exception that shipped. Multi-model second opinion paid
  for itself before it existed.
- **Fail-closed as a design language.** Every "can't verify" path (malformed
  settings, unreadable dotenv, moved config root) refuses instead of guessing.
  That made each new reviewer finding a one-shape fix.

## What was hard

- **Gemini's config surface.** Six of ten majors were gemini auth vectors:
  dotenv auto-load walk, persisted `selectedAuthType` (v1 flat + v2 nested)
  across home/workspace/system scopes, `GEMINI_CLI_SYSTEM_SETTINGS_PATH`,
  `GEMINI_DEFAULT_AUTH_TYPE`, `GEMINI_CLI_HOME`, ADC. A spawn-time preflight
  for a tool with that many config layers is whack-a-mole; we enumerated the
  documented surface, but future gemini CLI changes are the residual risk.
- **Score convergence.** The reviewer legitimately widened scope each round,
  so scores oscillated instead of climbing. What converged it: explicit
  calibration anchors in the prompt ("you judged this ship-ready at 8 before
  these fixes; consistency implies 9+") plus hard exclusions for
  out-of-scope hypotheticals.

## Do differently next time

- Budget review rounds by config-surface size of the touched dependency, not
  by feature size.
- Put score-calibration rules in the FIRST review prompt, not round 6.
- When one adapter accounts for most findings, consider descoping it to a
  follow-up rather than hardening in-loop — it was the right call to keep
  gemini here (it converged), but it was close.
