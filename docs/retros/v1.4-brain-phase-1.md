## Key decisions

- **Append-only is enforced by GRANTs, not triggers** — the runtime role has SELECT + column-scoped INSERT only; the admin/owner can insert freely. This is what makes id-preserving restore (`OVERRIDING SYSTEM VALUE`) clean.
- **Brokered DDL** — agent table creation goes through `brain.create_table()` (SECURITY DEFINER); agent tables have no separately-journaled schema, so backup captures their column specs from the catalog and import replays them through the same broker.
- **Append-only dedup/merge** (ROS-140) — merges write a merge-map row, resolved at read time via `brain.canonical_id()`; nothing is deleted. Codex caught a cycle-guard gap + a TOCTOU there, fixed with a full reachability walk + `pg_advisory_xact_lock`.
- **Backup (ROS-141)** — self-generated SQL (not `pg_dump`); JSONL is the canonical, verified, tool-restorable form; `--format sql` adds a standalone psql artifact import never executes; import hard-refuses a non-empty target and requires an exact schema-version match.

## Learnings to harvest

- The adversarial Codex review loop earns its keep: across the epic it caught real correctness bugs that tests alone missed — a merge cycle-guard gap, a guard/insert TOCTOU, manifest cast-injection, transaction-safety of opaque SQL replay, and a create-table TOCTOU. Budget for 2–4 review→fix rounds on DB-correctness work.
- Backup/restore for an append-only, id-as-audit store has non-obvious requirements: preserve ids (`OVERRIDING SYSTEM VALUE`), reset identity sequences (3-arg `setval`, empty-table case), encode every cell as `::text` and re-cast by type to keep SQL-NULL vs JSONB-null and bigint/numeric/microsecond fidelity, and serialize the table broker against the restore.
- "Self-generated SQL restored by the tool" is a trap — opaque SQL can't be verified or rolled back cleanly. JSONL-as-canonical + SQL-as-psql-artifact is the safe split.
